# ブルアカ「ストーリー・イベント内容(小ネタ)」作問ガイド

この文書は、AIにブルーアーカイブのストーリー・イベント内容(小ネタ)クイズを作らせるための依頼テンプレートです。
下の「依頼文テンプレート」をそのまま貼り付け、`○○○○` にストーリー名・章名・イベント名を入れて使ってください。

## 依頼文テンプレート

```txt
ブルーアーカイブのストーリー・イベント内容(小ネタ)クイズを作ってください。

対象ストーリー:
○○○○

参考サイト:
- ブルアカJPテキスト化wiki: https://wikiwiki.jp/ba_storytext/
- ブルーアーカイブ攻略 Wiki: https://bluearchive.wikiru.jp/
- bluearchive.wiki: https://bluearchive.wiki/
- Blue Archive Wiki Fandom: https://bluearchive.fandom.com/
- ぽむぜろアーカイブ: https://pomzero.hatenablog.com/archive/category/%E3%83%96%E3%83%AB%E3%83%BC%E3%82%A2%E3%83%BC%E3%82%AB%E3%82%A4%E3%83%96%EF%BC%9A%E3%82%B9%E3%83%88%E3%83%BC%E3%83%AA%E3%83%BC

最優先で参照するのは、台詞本文やエピソード本文が確認できる「ブルアカJPテキスト化wiki」です。
要約記事だけを参照すると重要シーン中心になり簡単になりやすいので、要約だけで作問しないでください。

作問ジャンル:
ストーリー・イベント内容(小ネタ)

問題数:
10問

作問方針:
- 要約に載る重要事件、目的、黒幕、結末、主要な因果関係は避ける
- キャラの恒常的な性格、所属、役職、口癖だけで解ける問題は避ける
- ストーリー名、イベント名、章タイトルから答えが連想できる問題は避ける
- 1エピソード内の一時的な描写、注文、地名、ブロック名、数値、ナンバー、台詞中の固有表現、小道具、軽いやり取りから出題する
- 「一度ストーリーを読んだ人でも、細部を流していたら落とす」難度を狙う
- ただし、理不尽な捏造問題にはしない。答えは参照本文に明確に存在するものにする
- 選択肢は同じカテゴリ、同じ文体、同じ粒度で揃える
- 誤答はそれらしく作ってよいが、正解と同じ種類の語にする
- 問題文に答えの単語や、答えを強く連想させる単語を入れない
- 「同イベントより出題」など、単独で表示した時に通じない省略はしない

良い問題の例:
- その場で注文したラーメンの種類
- 画面に表示された地区名やブロック名
- 会話中に出た具体的な日数、深さ、番号、ナンバープレート
- 一瞬だけ出た小道具や貼られていたもの
- 冗談として提案された副業、作戦名、チーム名
- 怪異や事件への対処として実際に取ったコミカルな行動

悪い問題の例:
- 黒幕は誰か
- 最終的に何を解決したか
- そのイベントの主役キャラは誰か
- アビドスの問題は何か
- 便利屋68はどんな部活か
- ホシノはどんな性格か
- タイトルに「夏」とあるイベントで海を答えさせる
- 「紛争調停委員会」の役割を答えさせる

出力形式:
JavaScriptの配列形式で出してください。
各問題は以下の形式にしてください。

[
  {
    genre: "ストーリー・イベント内容(小ネタ)",
    sourceTitle: "ストーリー名・章名・エピソード名",
    prompt: "問題文",
    answer: "正解",
    distractors: ["誤答1", "誤答2", "誤答3"]
  }
]

出力後に、各問題について以下を短く自己評価してください。
- 良問
- やや簡単
- 要確認
- 不採用候補
```

## 採用基準

採用する問題は、以下を満たすものを優先します。

- 本筋の要約だけでは解けない
- キャラ知識だけでは解けない
- ストーリー本文を読んでいれば答えが確認できる
- 選択肢だけを見ても正解が浮かない
- 誤答が正解と同じカテゴリで揃っている
- 問題文に答えのヒントを入れすぎていない

## 避けるべき問題

以下の問題は、難しそうに見えてもクイズとして弱いです。

- 章やイベントの中心テーマそのものを答えさせる問題
- 主要キャラ、所属、役職、学校名だけを答えさせる問題
- タイトルから連想できる答え
- 問題文と答えで同じ単語を共有している問題
- 4択のうち正解だけカテゴリが違う問題
- 誤答だけ明らかに不自然な問題

## 誤答作成ルール

誤答は、本文にない語を作っても構いません。
ただし、正解と同じカテゴリにしてください。

例:

- 正解が「し44-66」なら、誤答もナンバープレート風にする
- 正解が「8メートル」なら、誤答も深さの数値にする
- 正解が「調査班X」なら、誤答もチーム名風にする
- 正解が「替え玉追加」なら、誤答もラーメン店の追加注文にする
- 正解が「モー超常現象研究所」なら、誤答も動画チャンネル名・研究所名風にする

## このジャンルの役割

`ストーリー・イベント内容(小ネタ)` は、通常の `ストーリー内容` と分けて扱います。

- `ストーリー内容`: 章やイベントの流れ、出来事、因果関係を問う中級問題
- `ストーリー・イベント内容(小ネタ)`: 本文細部、会話、小道具、数値、注文、地名を問う上級問題

本番クイズでは、通常のストーリー内容より出題数を少なめにするとバランスが取りやすいです。

## 通常ジャンル「ストーリー内容」の作問メモ

`ストーリー内容` も、要約しすぎると簡単になります。
通常ジャンルであっても、prompt と answer はできるだけ本文の発言・描写に寄せてください。

避けるべき形:

```js
{
  prompt: "ヘルメット団の背後に別勢力がいると考えたきっかけは？",
  answer: "使用されていた装備の出どころ"
}
```

良い形:

```js
{
  prompt: "セリカ救出後、戦闘中に回収した戦車の部品を確認したアヤネは、それがどのようなものだと判明したと話した？",
  answer: "キヴォトスでは使用が禁止されている違法機種",
  distractors: [
    "カイザーPMCだけが運用を許可されている特殊機種",
    "ブラックマーケットで一般流通している旧式機種",
    "ゲヘナ風紀委員会から盗まれた正式配備機種"
  ]
}
```

`ストーリー内容` は「流れを知っていれば解ける」ではなく、「本文の具体的な発言・描写を理解していれば解ける」難度を狙います。
